Smart Routing — Waypoint Injection Design

How the algorithm detours around flood zones using turf.js geometry (no new dependencies needed)

❌ Current behavior
✓ With waypoint injection

Algorithm steps

Step 1 — Initial fetch

Call Mapbox with alternatives=true (already done). Run findSafestRoute. If best route is dry → done, return it.

Step 2 — Detect crossing

If best route is flooded, get the flood zone polygons it crosses. For each, use turf.nearestPointOnLine + perpendicular bearing to compute a bypass waypoint ~300m from flood center.

Step 3 — Re-route

Call Mapbox again with [origin, bypassWaypoint, destination]. Re-run findSafestRoute. One retry maximum.

Step 4 — Fallback

If still flooded after retry, return the route with isFlooded: true and a clear message: "Only available route crosses a flood zone."

All turf.js — no new dependencies

Uses turf.nearestPointOnLine, turf.bearing, and turf.destination — all already available since @turf/turf is imported in routingService.js.
